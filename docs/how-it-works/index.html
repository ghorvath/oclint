How It Works

OCLint checks source code against rules, and produces a report.

  1. Pass a file name and a ruleset (by default, could be dynamically loaded into the program) into OCLint
  2. OCLint hands the input file off to parser in libclang
  3. OCLint gets an Abstract Syntax Tree (AST) from libclang
  4. Traverse AST, for each of the node, apply every rule in the ruleset and check for problems. (Please notice that AST only be traversed once and apply rules to each node.)
  5. The violations are now stored in the ViolationSet, and OCLint will choose a reporter to render output.
